Detailed Description

Some functions require special processing, to ensure that events are sent in the right order and then processed before returning. If the event is for a window within this application, a message loop may be needed to ensure that the callback is completed before the function returns. These functions wrap the X11 functions in which this sort of synchronous processing is required.
You should ALWAYS call the 'X11 Workbench' versions, rather than the libX11 versions, to ensure that events happen in the right order.

Invokes the functionality of 'XConvertSelection' in a safe and proper manner.

Parameters
pDisplayThe Display pointer, or 'None' to use default
wIDOwnerThe owner window of the selection, if known ('None' to auto-discover)
aSelectionThe 'Selection' atom, identifying the selection (typically CLIPBOARD)
aTargetThe 'Target' atom, destination after conversion
aPropertyThe 'Property' atom, source for conversion
tmEventThe 'event time', typically CurrentTime or WBGetLastEventTime()

Use this function in lieu of direct calls to 'XConvertSelection'. It will use the hidden (internal) window as the requestor and wait for a response. If the clipboard target is owned by this application, internal functions will be used for the property request/notification events.
