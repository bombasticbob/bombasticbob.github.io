403 // The ICCCM provides for focus models for a window:
404 //
405 // Input Model Input Field WM_TAKE_FOCUS
406 // No Input False Absent
407 // Passive True Absent
408 // Locally Active True Present
409 // Globally Active False Present
410 //
411 // No Input - The application never expects to receive focus.
412 //
413 // Passive - The application will get focus but it will not grab it on
414 // its own. Instead it will only get focus if the window manager gives it
415 // to it. The application cannot help determine if it wants the focus or
416 // not at the given time/situation.
417 //
418 // Locally Active - The same as passive, but the application will also
419 // give focus to other windows that it owns of its own free will, without
420 // any interaction with the window manager, using XSetInputFocus.
421 //
422 // Globally Active - The application will not receive focus from the
423 // window manager. Instead it will determine when it wants focus (from
424 // WM_TAKE_FOCUS suggestions given by the window manager, or from events
425 // such as ButtonPress or KeyPress). The application will then acquire
426 // focus on its own using XSetInputFocus.
